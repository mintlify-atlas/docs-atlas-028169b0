The analytics and reporting surface of the Workforce Intelligence OS is split into four layers: per-user employee dashboards, manager team views, admin org-wide health panels, and a dedicated analytics router for deeper performance signals. Every endpoint is protected by a valid JWT and enforces role or permission checks before returning data. All paths are prefixed with `/api/v1`.

<Note>
  All analytics endpoints require an `Authorization: Bearer <token>` header. Requests without a valid token receive `401 AUTH_ERROR`.
</Note>

## Employee Dashboard

### `GET /api/v1/dashboard/employee`

Returns KPI cards and live session state for the currently authenticated employee. Any active role may call this endpoint.

**Auth:** Any authenticated user (`get_current_user`).

<ResponseField name="attendance" type="object" required>
  Current shift attendance summary.
  <Expandable title="AttendanceSummary fields">
    <ResponseField name="checked_in_today" type="boolean">Whether the user has checked in today.</ResponseField>
    <ResponseField name="check_in_at" type="string | null">ISO-8601 UTC timestamp of check-in.</ResponseField>
    <ResponseField name="check_out_at" type="string | null">ISO-8601 UTC timestamp of check-out.</ResponseField>
    <ResponseField name="work_mode" type="string | null">`OFFICE` or `WFH`.</ResponseField>
    <ResponseField name="session_status" type="string | null">Current session status enum value.</ResponseField>
    <ResponseField name="duration_minutes" type="integer | null">Total session duration so far in minutes.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tasks" type="object" required>
  Aggregated task counts for the user.
  <Expandable title="TaskCounts fields">
    <ResponseField name="total" type="integer">All assigned tasks.</ResponseField>
    <ResponseField name="in_progress" type="integer">Tasks currently in progress.</ResponseField>
    <ResponseField name="completed" type="integer">Completed tasks.</ResponseField>
    <ResponseField name="blocked" type="integer">Blocked tasks.</ResponseField>
    <ResponseField name="overdue" type="integer">Past due-date and not yet completed or reviewed.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="timer" type="object" required>
  Active time-log state.
  <Expandable title="TimerState fields">
    <ResponseField name="active" type="boolean">Whether a timer is currently running.</ResponseField>
    <ResponseField name="task_id" type="string | null">UUID of the task being timed.</ResponseField>
    <ResponseField name="task_title" type="string | null">Title of the active task.</ResponseField>
    <ResponseField name="started_at" type="string | null">ISO-8601 UTC start timestamp.</ResponseField>
    <ResponseField name="elapsed_minutes" type="integer | null">Minutes elapsed since start.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="attendance_status" type="string">Human-readable session status, e.g. `"checked_in"` or `"not_checked_in"`.</ResponseField>
<ResponseField name="total_time_today" type="integer">Total session minutes today.</ResponseField>
<ResponseField name="productive_time_today" type="integer">Total logged productive minutes today.</ResponseField>
<ResponseField name="tasks_in_progress" type="integer">Count of tasks currently in-progress.</ResponseField>
<ResponseField name="tasks_due_soon" type="integer">Tasks due within 2 days.</ResponseField>

```json Example Response
{
  "attendance": {
    "checked_in_today": true,
    "check_in_at": "2025-01-20T09:02:14Z",
    "check_out_at": null,
    "work_mode": "OFFICE",
    "session_status": "checked_in",
    "duration_minutes": 127
  },
  "tasks": {
    "total": 12,
    "in_progress": 3,
    "completed": 7,
    "blocked": 1,
    "overdue": 1
  },
  "timer": {
    "active": true,
    "task_id": "b3a1c2d4-...",
    "task_title": "Implement login flow",
    "started_at": "2025-01-20T10:15:00Z",
    "elapsed_minutes": 32
  },
  "attendance_status": "checked_in",
  "total_time_today": 127,
  "productive_time_today": 95,
  "tasks_in_progress": 3,
  "tasks_due_soon": 2
}
```

## Manager Dashboard

### `GET /api/v1/dashboard/manager`

Returns team attendance, pending approval counts, and overdue/blocked task summaries for the authenticated manager. Admins calling this endpoint see all active users as their "team."

**Auth:** `MANAGER` or `ADMIN` role required.

<ResponseField name="team_attendance_today" type="array">
  List of team member attendance records for today.
  <Expandable title="TeamMemberAttendance fields">
    <ResponseField name="user_id" type="string">UUID of the team member.</ResponseField>
    <ResponseField name="full_name" type="string">Display name.</ResponseField>
    <ResponseField name="checked_in" type="boolean">Whether they have an active session.</ResponseField>
    <ResponseField name="work_mode" type="string | null">`OFFICE` or `WFH`.</ResponseField>
    <ResponseField name="check_in_at" type="string | null">ISO-8601 UTC check-in time.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pending_approvals_count" type="integer">Total pending leave requests + attendance corrections awaiting this manager.</ResponseField>
<ResponseField name="overdue_tasks_count" type="integer">Overdue tasks across the team.</ResponseField>
<ResponseField name="blocked_tasks_count" type="integer">Blocked tasks across the team.</ResponseField>
<ResponseField name="team_members_active" type="integer">Number of team members currently checked in.</ResponseField>

## Analytics Signals

Returns a ranked list of top-performing employees based on task completion and attendance consistency. Requires `analytics.view_org` permission.

**Auth:** `analytics.view_org` permission required.

<ResponseField name="[]" type="array">
  Array of `BestPerformer` objects.
  <Expandable title="BestPerformer fields">
    <ResponseField name="user_id" type="string">UUID of the employee.</ResponseField>
    <ResponseField name="full_name" type="string" />
    <ResponseField name="score" type="number">Composite performance score.</ResponseField>
    <ResponseField name="completed_tasks" type="integer">Count of completed tasks in the scoring window.</ResponseField>
    <ResponseField name="attendance_consistency" type="number">Attendance consistency score (0–1).</ResponseField>

### `GET /api/v1/analytics/burnout-risks`

Returns employees exhibiting burnout risk signals based on consecutive high-hour working days. Requires `analytics.view_org` permission.

**Auth:** `analytics.view_org` permission required.

<ResponseField name="[]" type="array">
  Array of `BurnoutRisk` objects.
  <Expandable title="BurnoutRisk fields">
    <ResponseField name="user_id" type="string">UUID of the employee.</ResponseField>
    <ResponseField name="full_name" type="string" />
    <ResponseField name="consecutive_high_hour_days" type="integer">Days in a row with above-threshold logged hours.</ResponseField>
    <ResponseField name="risk_level" type="string">One of `Low`, `Medium`, `High`.</ResponseField>
  </Expandable>
</ResponseField>

<Warning>
  This endpoint only returns employees flagged above the low-risk threshold. Employees with no risk signals are omitted from the response.
</Warning>

### `GET /api/v1/reports/team-performance`

Returns a paginated, filterable team performance report. Supports date range anchoring via a `period` mode or explicit `start_date`/`end_date`.

**Auth:** `MANAGER`, `HR_OPERATIONS`, or `ADMIN` role required.

<ParamField query="period" type="string">
  One of `daily`, `weekly`, `monthly`, or `custom`. Defaults to `weekly`.
</ParamField>

<ParamField query="date" type="string">
  Anchor date for daily/weekly/monthly periods. ISO format `YYYY-MM-DD`.
</ParamField>

<ParamField query="start_date" type="string">
  Explicit start date for `custom` period. ISO format `YYYY-MM-DD`.
</ParamField>

<ParamField query="end_date" type="string">
  Explicit end date for `custom` period. ISO format `YYYY-MM-DD`.
</ParamField>

<ParamField query="search" type="string">
  Filter by employee name, email, role, department, or designation.
</ParamField>

<ParamField query="department_id" type="string">
  Filter by department UUID.
</ParamField>

<ParamField query="role" type="string">
  Filter by role string.
</ParamField>

<ParamField query="page" type="integer">
  Page number (1-based). Omit for unpaginated results.
</ParamField>

<ParamField query="page_size" type="integer">
  Rows per page. Min `1`, max `200`.
</ParamField>

<ResponseField name="start_date" type="string">Effective start date of the report period.</ResponseField>
<ResponseField name="end_date" type="string">Effective end date of the report period.</ResponseField>
<ResponseField name="rows" type="array">Per-employee rows with hours, task counts, attendance breakdowns, and EOD status.</ResponseField>

```bash Example Request
GET /api/v1/reports/team-performance?period=weekly&date=2025-01-20&page=1&page_size=25
Authorization: Bearer <token>
```

### `GET /api/v1/reports/team-performance/export`

Streams the team performance report as a CSV file. Accepts the same filter parameters as `team-performance` but returns all rows (no pagination). The response has `Content-Type: text/csv` and a `Content-Disposition: attachment` header.

**Auth:** `MANAGER`, `HR_OPERATIONS`, or `ADMIN` role required.

<ParamField query="period" type="string">One of `daily`, `weekly`, `monthly`, `custom`. Defaults to `weekly`.</ParamField>
<ParamField query="date" type="string">Anchor date ISO `YYYY-MM-DD`.</ParamField>
<ParamField query="start_date" type="string">Explicit start date for custom period.</ParamField>
<ParamField query="end_date" type="string">Explicit end date for custom period.</ParamField>
<ParamField query="search" type="string">Name/email/role filter.</ParamField>
<ParamField query="department_id" type="string">Filter by department UUID.</ParamField>

## Error Responses

All analytics and reports endpoints share the standard error envelope:

```json
{
  "error": {
    "code": "PERMISSION_ERROR",
    "message": "Admin only",
    "details": []
  }
}
```

| HTTP Status | Code | Reason |
|---|---|---|
| `401` | `AUTH_ERROR` | Missing or invalid JWT |
| `403` | `PERMISSION_ERROR` | Insufficient role or permission |
| `422` | `VALIDATION_ERROR` | Invalid date range or query parameters |
| `500` | `INTERNAL_ERROR` | Service computation failure |
