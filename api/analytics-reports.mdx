The analytics and reporting surface of the Workforce Intelligence OS is split into three layers: per-user employee dashboards, manager team views, and admin org-wide health panels. Every endpoint is protected by a valid JWT and enforces role or permission checks before returning data. All paths are prefixed with `/api/v1`.

<Note>
  All analytics endpoints require an `Authorization: Bearer <token>` header. Requests without a valid token receive `401 AUTH_ERROR`.
</Note>

## Employee Dashboard

### `GET /api/v1/dashboard/employee`

Returns KPI cards and live session state for the currently authenticated employee. Any active role may call this endpoint.

**Auth:** Any authenticated user (`get_current_user`)

<ResponseField name="attendance" type="object" required>
  Current shift attendance summary.
  <Expandable title="AttendanceSummary fields">
    <ResponseField name="checked_in_today" type="boolean">Whether the user has checked in today.</ResponseField>
    <ResponseField name="check_in_at" type="string | null">ISO-8601 UTC timestamp of check-in.</ResponseField>
    <ResponseField name="check_out_at" type="string | null">ISO-8601 UTC timestamp of check-out.</ResponseField>
    <ResponseField name="work_mode" type="string | null">`OFFICE` or `WFH`.</ResponseField>
    <ResponseField name="session_status" type="string | null">Current session status enum value.</ResponseField>
    <ResponseField name="duration_minutes" type="integer | null">Total session duration so far in minutes.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tasks" type="object" required>
  Aggregated task counts for the user.
  <Expandable title="TaskCounts fields">
    <ResponseField name="total" type="integer">All assigned tasks.</ResponseField>
    <ResponseField name="in_progress" type="integer">Tasks currently in progress.</ResponseField>
    <ResponseField name="completed" type="integer">Completed tasks.</ResponseField>
    <ResponseField name="blocked" type="integer">Blocked tasks.</ResponseField>
    <ResponseField name="overdue" type="integer">Past due-date and not yet completed or reviewed.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="timer" type="object" required>
  Active time-log state.
  <Expandable title="TimerState fields">
    <ResponseField name="active" type="boolean">Whether a timer is currently running.</ResponseField>
    <ResponseField name="task_id" type="string | null">UUID of the task being timed.</ResponseField>
    <ResponseField name="task_title" type="string | null">Title of the active task.</ResponseField>
    <ResponseField name="started_at" type="string | null">ISO-8601 UTC start timestamp.</ResponseField>
    <ResponseField name="elapsed_minutes" type="integer | null">Minutes elapsed since start.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="attendance_status" type="string">Human-readable session status, e.g. `"checked_in"` or `"not_checked_in"`.</ResponseField>
<ResponseField name="total_time_today" type="integer">Total session minutes today.</ResponseField>
<ResponseField name="productive_time_today" type="integer">Total logged productive minutes today.</ResponseField>
<ResponseField name="tasks_in_progress" type="integer">Count of tasks currently in-progress.</ResponseField>
<ResponseField name="tasks_due_soon" type="integer">Tasks due within 2 days.</ResponseField>

```json Example Response
{
  "attendance": {
    "checked_in_today": true,
    "check_in_at": "2025-01-20T09:02:14Z",
    "check_out_at": null,
    "work_mode": "OFFICE",
    "session_status": "checked_in",
    "duration_minutes": 127
  },
  "tasks": {
    "total": 12,
    "in_progress": 3,
    "completed": 7,
    "blocked": 1,
    "overdue": 1
  },
  "timer": {
    "active": true,
    "task_id": "b3a1c2d4-...",
    "task_title": "Implement login flow",
    "started_at": "2025-01-20T10:15:00Z",
    "elapsed_minutes": 32
  },
  "attendance_status": "checked_in",
  "total_time_today": 127,
  "productive_time_today": 95,
  "tasks_in_progress": 3,
  "tasks_due_soon": 2
}
```

## Manager Dashboard

### `GET /api/v1/dashboard/manager`

Returns team attendance, pending approval counts, and overdue/blocked task summaries for the authenticated manager. Admins calling this endpoint see all active users as their "team."

**Auth:** `MANAGER` or `ADMIN` role required.

<ResponseField name="team_attendance_today" type="array">
  List of team member attendance records for today.
  <Expandable title="TeamMemberAttendance fields">
    <ResponseField name="user_id" type="string">UUID of the team member.</ResponseField>
    <ResponseField name="full_name" type="string">Display name.</ResponseField>
    <ResponseField name="checked_in" type="boolean">Whether they have an active session.</ResponseField>
    <ResponseField name="work_mode" type="string | null">`OFFICE` or `WFH`.</ResponseField>
    <ResponseField name="check_in_at" type="string | null">ISO-8601 UTC check-in time.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pending_approvals_count" type="integer">Total pending leave requests + attendance corrections awaiting this manager.</ResponseField>
<ResponseField name="overdue_tasks_count" type="integer">Overdue tasks across the team.</ResponseField>
<ResponseField name="blocked_tasks_count" type="integer">Blocked tasks across the team.</ResponseField>
<ResponseField name="team_members_active" type="integer">Number of team members currently checked in.</ResponseField>

```json Example Response
{
  "team_attendance_today": [
    {
      "user_id": "a1b2c3d4-...",
      "full_name": "Fatima Malik",
      "checked_in": true,
      "work_mode": "WFH",
      "check_in_at": "2025-01-20T09:10:00Z"
    }
  ],
  "pending_approvals_count": 4,
  "overdue_tasks_count": 2,
  "blocked_tasks_count": 1,
  "team_members_active": 7,
  "pending_approvals": 4,
  "overdue_tasks": 2,
  "blocked_tasks": 1
}
```

## Admin Dashboard

### `GET /api/v1/dashboard/admin`

Returns organisation-wide health metrics including headcount, attendance breakdown, open alerts, and project counts.

**Auth:** `ADMIN` role required.

<ResponseField name="total_users" type="integer">Total registered users.</ResponseField>
<ResponseField name="active_users" type="integer">Users with `ACTIVE` status.</ResponseField>
<ResponseField name="checked_in_today" type="integer">Employees currently checked in.</ResponseField>
<ResponseField name="wfh_today" type="integer">Employees working from home today.</ResponseField>
<ResponseField name="office_today" type="integer">Employees in the office today.</ResponseField>
<ResponseField name="pending_approvals_count" type="integer">Total org-wide pending approval decisions.</ResponseField>
<ResponseField name="open_alerts_count" type="integer">Number of unresolved system alerts.</ResponseField>
<ResponseField name="overdue_tasks_count" type="integer">Org-wide overdue task count.</ResponseField>
<ResponseField name="active_projects" type="integer">Projects with `ACTIVE` or `APPROVED` status.</ResponseField>

```json Example Response
{
  "total_users": 84,
  "active_users": 78,
  "checked_in_today": 51,
  "wfh_today": 14,
  "office_today": 37,
  "pending_approvals_count": 9,
  "open_alerts_count": 3,
  "overdue_tasks_count": 11,
  "active_projects": 6
}
```

## Admin Analytics Dashboard

### `GET /api/v1/dashboard/admin/analytics`

Delivers a rich analytics payload for the admin dashboard: KPIs, 7-day attendance trends, task and project statistics, per-department comparison, people exceptions, and recent activity.

**Auth:** `ADMIN` role required.

<ResponseField name="kpis" type="object">
  Top-level key performance indicators.
  <Expandable title="AdminAnalyticsKPIs fields">
    <ResponseField name="total_employees" type="integer" />
    <ResponseField name="total_projects" type="integer" />
    <ResponseField name="pending_approvals" type="integer" />
    <ResponseField name="attendance_rate" type="number">Floating-point percentage (0–100).</ResponseField>
    <ResponseField name="checked_in_today" type="integer" />
    <ResponseField name="late_today" type="integer" />
    <ResponseField name="wfh_today" type="integer" />
  </Expandable>
</ResponseField>

<ResponseField name="attendance_trend" type="array">
  Daily attendance data points (typically last 7 days).
  <Expandable title="AdminAnalyticsAttendanceTrend fields">
    <ResponseField name="date" type="string">ISO date string `YYYY-MM-DD`.</ResponseField>
    <ResponseField name="checked_in" type="integer" />
    <ResponseField name="late" type="integer" />
    <ResponseField name="absent" type="integer" />
  </Expandable>
</ResponseField>

<ResponseField name="task_statistics" type="object">Counts by status: `total`, `completed`, `in_progress`, `on_hold`, `pending`, `rejected`.</ResponseField>
<ResponseField name="project_statistics" type="object">Counts by status: `total`, `approved`, `pending`, `rejected`, `active`.</ResponseField>
<ResponseField name="department_comparison" type="array">Per-department attendance rate and task completion metrics.</ResponseField>
<ResponseField name="people_exceptions" type="array">Employees flagged for exceptions (e.g. chronic late arrivals, blockers).</ResponseField>
<ResponseField name="recent_activity" type="array">Latest system-wide activity log entries.</ResponseField>

<Tip>
  Use this endpoint to power a scrollable admin command-centre panel. Cache responses on the frontend for up to 60 seconds to reduce database load.
</Tip>

## Attendance Reports

### `GET /api/v1/reports/attendance`

Returns team or org-wide attendance aggregations for a date range. The response content depends on the caller's role. Manager/HR users see their scoped team; Admins see the full organisation.

**Auth:** `MANAGER`, `HR_OPERATIONS`, or `ADMIN` role required.

<ParamField query="start_date" type="string" required>
  Report start date — ISO format `YYYY-MM-DD`.
</ParamField>

<ParamField query="end_date" type="string" required>
  Report end date — ISO format `YYYY-MM-DD`.
</ParamField>

<ResponseField name="[]" type="array">
  Array of `WeeklyReportRead` objects — one per scoped team member.
  <Expandable title="WeeklyReportRead fields">
    <ResponseField name="user_id" type="string">UUID of the employee.</ResponseField>
    <ResponseField name="full_name" type="string" />
    <ResponseField name="attendance_days" type="integer">Days with a completed attendance session.</ResponseField>
    <ResponseField name="late_days" type="integer" />
    <ResponseField name="wfh_days" type="integer" />
    <ResponseField name="total_logged_hours" type="number" />
  </Expandable>
</ResponseField>

```bash Example Request
GET /api/v1/reports/attendance?start_date=2025-01-13&end_date=2025-01-19
Authorization: Bearer <token>
```

<Note>
  A separate **export** endpoint at `GET /api/v1/reports/team-performance/export` streams a CSV file of the same data with query parameters `period`, `start_date`, `end_date`, `department_id`, and `role`.
</Note>

## Growth Metrics

### `GET /api/v1/growth/goals`

Lists personal goals for the authenticated user.

### `GET /api/v1/growth/achievements`

Lists earned achievement badges for the authenticated user.

**Auth:** Any authenticated user.

<ResponseField name="[]" type="array">
  Array of achievement objects.
  <Expandable title="AchievementRead fields">
    <ResponseField name="id" type="string">UUID.</ResponseField>
    <ResponseField name="user_id" type="string" />
    <ResponseField name="badge_name" type="string" />
    <ResponseField name="title" type="string" />
    <ResponseField name="description" type="string | null" />
    <ResponseField name="icon_name" type="string | null" />
    <ResponseField name="date" type="string">ISO-8601 datetime of award.</ResponseField>
  </Expandable>
</ResponseField>

### `GET /api/v1/growth/team`

Returns growth and achievement records for the manager's direct reports (or all employees for Admins).

**Auth:** Any authenticated user — result scope is role-dependent.


## Error Responses

All analytics endpoints share the standard error envelope:

```json
{
  "error": {
    "code": "PERMISSION_ERROR",
    "message": "Admin only",
    "details": []
  }
}
```

| HTTP Status | Code | Reason |
|---|---|---|
| `401` | `AUTH_ERROR` | Missing or invalid JWT |
| `403` | `PERMISSION_ERROR` | Insufficient role |
| `500` | `INTERNAL_ERROR` | Service computation failure |
